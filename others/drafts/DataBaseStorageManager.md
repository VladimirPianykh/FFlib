# Техническое задание

## Постановка
Разработать класс `DataBaseStorageManager`, реализующий интерфейс `StorageManager`.  
Класс должен мочь отправлять `SELECT`-запросы в базу для агрегации данных из EditableGroup. Также он предоставляет модели и `Saver`ы для всех `Feature`, хранилище — реализацию `Data` — и реализацию `UserSaver`.

### Функции `DataBaseStorageManager` (помимо контракта `StorageManager`)
- `ResultSet fetch(String sql)` - выполняет `SELECT`-запрос. Проверяет, что запрос read-only. Использует кэширование.

## Требования к технологиям
- Драйвер БД — h2
- JDBC

## Структура
`DataBaseStorageManager` должен поддерживать соединение с базой за счёт внутреннего класса `DataBaseBridge`.
`FeatureSaver`ы, `FeatureModel`ы хранятся в реестре `FunctionalRegistry` внутри класса.

Подробнее о важных классах.

### `DataBaseBridge`
Поддерживает соединение с базой, отправляет запросы, при необходимости восстанавливает соединение.

### `DataBaseUserSaver`
Сохраняет пользователей в базе с указанием версии.

### `DataBaseData`
Хранит произвольные переменные и данные в `EditableGroup` в базе. Остальное чисто в памяти.
